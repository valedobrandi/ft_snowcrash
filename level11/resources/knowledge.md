Lua Server Command Injection

The Lua server listens on 127.0.0.1:5151 and asks the client for a password. The input is passed to the function hash(pass), which executes the shell command echo <password> | sha1sum using io.popen(). Because the password is concatenated directly into a shell command, it creates a command injection vulnerability. An attacker can include shell separators like ; to execute additional commands. For example, sending test; getflag causes the shell to run echo test; getflag | sha1sum. The getflag command executes with the privileges of the service (flag11). This allows the attacker to retrieve the flag even though the password hash check fails.
